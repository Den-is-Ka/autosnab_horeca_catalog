# АвтоСнаб HoReCa Catalog

Адаптивный каталог продуктов для предприятий HoReCa с общей корзиной и AI-помощником на базе CopilotKit.

AI-помощник подбирает товары под заданный бюджет и отображает рекомендации как интерактивные React-карточки непосредственно внутри чата. Товары, добавленные из AI-чата, попадают в ту же корзину, что и товары из основного каталога.

## Возможности проекта

- каталог из 8 товаров HoReCa;
- локальное хранение данных в JSON;
- адаптивная сетка карточек;
- локальные изображения товаров;
- отображение названия, поставщика, цены и единицы измерения;
- выбор количества товара;
- автоматический расчёт стоимости выбранного количества;
- добавление товара в общую корзину;
- суммарный счётчик товаров в шапке;
- уведомление после добавления товара;
- AI-помощник в `CopilotSidebar`;
- локальный AI mock без внешнего API-ключа;
- интерактивные рекомендации внутри чата;
- добавление товара из AI-карточки в общую корзину;
- подтверждение добавления непосредственно в чате;
- состояния загрузки, ошибки и пустого каталога;
- автоматические тесты;
- запуск через Docker Compose.

## Демонстрационный AI-сценарий

Откройте AI-помощника и отправьте запрос:

```text
Подбери товары на сумму до 5 000 Р
```

AI-помощник предложит четыре позиции:

| Товар | Количество | Цена |
|---|---:|---:|
| Филе куриное | 4 кг | 1 400 Р |
| Сыр для пиццы | 3 кг | 1 950 Р |
| Томаты | 5 кг | 1 050 Р |
| Сливки 33% | 2 шт. | 560 Р |
| **Итого** |  | **4 960 Р** |

Каждая рекомендация выводится как интерактивная React-карточка с кнопкой **«Добавить в корзину»**.

После нажатия кнопки:

1. товар добавляется в общий `CartContext`;
2. счётчик корзины в шапке обновляется;
3. внутри AI-чата появляется подтверждение добавления.

## Архитектура AI-сценария

```text
Пользователь
    ↓
CopilotSidebar
    ↓
POST /api/copilotkit
    ↓
CopilotKit Runtime
    ↓
BuiltInAgent
    ↓
OpenAI-compatible endpoint
    ↓
@copilotkit/aimock
    ↓
show_product_recommendations
    ↓
Интерактивные React-карточки
    ↓
CartContext
    ↓
Общий счётчик корзины
```

Сообщение пользователя проходит через CopilotKit Runtime.

Рекомендации не формируются обычной проверкой строки внутри React-компонента. Mock-агент возвращает вызов frontend tool:

```text
show_product_recommendations
```

Инструмент зарегистрирован через `useFrontendTool` и отрисовывает интерактивный пользовательский интерфейс внутри чата.

## Требования для локального запуска

- Node.js 24;
- npm;
- свободный порт `3001` для Next.js;
- свободный порт `4010` для aimock.

Внешний API-ключ OpenAI, Anthropic или Gemini не требуется.

## Локальный запуск

### 1. Установите зависимости

```bash
npm ci
```

### 2. Запустите AI mock

В первом терминале:

```bash
npx @copilotkit/aimock --config ./aimock.json
```

Ожидаемый вывод:

```text
aimock server listening on http://127.0.0.1:4010
```

Оставьте этот терминал открытым.

### 3. Запустите Next.js

Во втором терминале:

```bash
npm run dev -- -p 3001
```

Откройте приложение:

```text
http://localhost:3001
```

## Переменные окружения

Проект имеет рабочие значения по умолчанию и может запускаться без `.env.local`.

При необходимости создайте `.env.local` на основе `.env.example`:

```dotenv
AIMOCK_BASE_URL=http://127.0.0.1:4010/v1
COPILOTKIT_TELEMETRY_DISABLED=true
```

Для запуска через Docker Compose адрес aimock задаётся автоматически:

```text
http://aimock:4010/v1
```

## Запуск через Docker Compose

Docker Compose запускает два связанных сервиса:

- `aimock` — локальный OpenAI-compatible mock-сервер;
- `app` — Next.js-приложение с CopilotKit Runtime.

Соберите и запустите проект:

```bash
docker compose up --build -d
```

Проверьте состояние контейнеров:

```bash
docker compose ps
```

Оба контейнера должны перейти в состояние `healthy`:

```text
autosnab-horeca-aimock
autosnab-horeca-catalog
```

Откройте приложение:

```text
http://localhost:3001
```

Логи AI mock:

```bash
docker compose logs --tail 100 aimock
```

Логи Next.js:

```bash
docker compose logs --tail 100 app
```

Остановка и удаление контейнеров:

```bash
docker compose down
```

## Состояния интерфейса

Каталог поддерживает следующие состояния:

- `loading state` — восемь skeleton-карточек;
- `error state` — сообщение об ошибке и кнопка повторной загрузки;
- `empty state` — сообщение об отсутствии товаров;
- обычное состояние с карточками товаров.

AI-инструмент поддерживает:

- состояние формирования подборки;
- интерактивные карточки рекомендаций;
- сообщение при невозможности сопоставить товар;
- подтверждение добавления товара в корзину.

## Адаптивность

Сетка каталога перестраивается в зависимости от ширины экрана:

- мобильные устройства — 1 карточка в строке;
- планшеты — 2 карточки в строке;
- широкие экраны — 4 карточки в строке.

Шапка, счётчик корзины и кнопка AI-помощника также адаптируются под ширину экрана.

## Особенности mock-режима

В проекте намеренно используется локальный `@copilotkit/aimock`.

Это позволяет:

- запускать проект без секретного API-ключа;
- передавать проект проверяющему без дополнительных настроек;
- получать воспроизводимый результат обязательного сценария;
- не зависеть от доступности внешнего LLM-провайдера;
- сохранить архитектуру CopilotKit Runtime;
- использовать frontend tools и Generative UI;
- показывать интерактивные React-компоненты внутри чата.

Конфигурация mock-сервера находится в:

```text
aimock.json
```

Основной fixture находится в:

```text
fixtures/horeca_assistant.json
```

Mock ориентирован на обязательный демонстрационный запрос. Для остальных сообщений AI-помощник возвращает подсказку с корректной формулировкой.

## Ограничения демонстрационной версии

- отдельная страница корзины не предусмотрена заданием;
- корзина хранится в памяти браузера;
- после перезагрузки страницы корзина очищается;
- mock-агент использует заранее подготовленный воспроизводимый сценарий;
- изображения являются локальными стилизованными иллюстрациями;
- свободный диалог с внешней LLM в демонстрационной версии не используется.
